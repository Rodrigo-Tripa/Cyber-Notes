# Web Architecture

## Overview

Web architecture describes how the different components of a web application communicate to deliver content to users. A typical request begins with a browser resolving a domain name through DNS, sending an HTTP request to a web server, and receiving the resources required to render the webpage.

## Components

Modern web applications commonly include **Web Servers**, **Application Servers**, **Databases**, **Load Balancers**, **Content Delivery Networks (CDNs)**, and **Web Application Firewalls (WAFs)**. Each component has a specific role, from serving content and storing data to improving performance, scalability, and security.

## Static vs Dynamic Content

Static websites deliver files exactly as they are stored on the server, while dynamic websites generate content at request time using server-side code and databases. Most modern web applications are dynamic, allowing them to provide authentication, personalized content, and interactive features.
