# Glossary

## CVE (Common Vulnerabilities and Exposures)

A standardized identifier assigned to publicly disclosed security vulnerabilities. Each CVE provides a unique reference that security professionals can use when discussing or researching a specific vulnerability.

## Exploit

Code or a technique that takes advantage of a vulnerability to achieve unintended behavior, such as gaining unauthorized access or executing arbitrary code.

## Google Dork

A search query that uses advanced search operators to discover specific information indexed by search engines. Google Dorks are commonly used for OSINT and security research.

## Hash

A fixed-length value generated from data using a hashing algorithm. Hashes are commonly used to verify data integrity and identify files.

## OSINT (Open Source Intelligence)

The process of collecting information from publicly available sources such as websites, search engines, social media, public records, and search databases.

## Proof of Concept (PoC)

A demonstration showing that a vulnerability can be successfully exploited. A PoC is intended to prove the existence of a security issue and is not necessarily a complete exploit.

## Search Engine

A service that indexes and retrieves information from the Internet based on user queries.

## Search Operator

A special command used to refine search engine results. Operators such as `site:` or `filetype:` help locate specific information more efficiently.

## Vulnerability

A weakness in software, hardware, or system configuration that could be exploited to compromise confidentiality, integrity, or availability.

## Vulnerability Database

A repository that stores information about known vulnerabilities, including identifiers, affected software, severity, and references.
