## Inside a Computer

Learned the basic components of a computer, how they work together, and the boot process from powering on the computer to loading the operating system.

---

## Computer Types

Learned about the main categories of computers and their different purposes. The room covered desktops, laptops, servers, smartphones, tablets, embedded systems, and mainframes, explaining how each is designed for specific workloads and environments. It also introduced the differences between personal computers and enterprise systems, highlighting factors such as performance, portability, scalability, and reliability.

---

## Client-Server Model

Learned how the client-server model works, where a client sends requests to a server, which processes them and returns a response. This architecture is the foundation of most modern networks and internet services.

Also learned that a single server can provide services to many clients simultaneously, and that protocols such as HTTP, HTTPS, SSH, FTP, and DNS enable communication between clients and servers.

---

## Virtualisation Basics

Learned what virtualization is and how it allows multiple virtual machines (VMs) to run independently on the same physical computer through a hypervisor. Each virtual machine has its own operating system, virtual hardware, and isolated environment while sharing the host's physical resources.

The room explained the difference between **Type 1 (bare-metal)** and **Type 2 (hosted)** hypervisors, as well as concepts such as snapshots, virtual disks, virtual networking, and resource allocation. It also covered the advantages of virtualization, including isolation, efficient hardware utilization, simplified testing, and why VMs are widely used in cybersecurity labs, malware analysis, penetration testing, and safe experimentation.

---

## Cloud Computing Fundamentals

Learned what cloud computing is and how organizations use remote infrastructure instead of maintaining their own physical hardware. Cloud providers deliver computing resources such as virtual machines, storage, databases, networking, and applications over the internet.

The room introduced the three main service models: **Infrastructure as a Service (IaaS)**, **Platform as a Service (PaaS)**, and **Software as a Service (SaaS)**. It also explained the differences between **public**, **private**, and **hybrid** clouds, along with key advantages such as scalability, high availability, flexibility, reduced maintenance, and cost efficiency through pay-as-you-go pricing.

---

## Operating Systems Introduction

Learned that an operating system is responsible for managing hardware resources and providing an interface between users, applications, and the computer's hardware. It controls processes, memory, storage, devices, user accounts, and permissions, allowing software to run efficiently.

The room introduced the role of the kernel, the differences between graphical user interfaces (GUI) and command-line interfaces (CLI), and compared common operating systems such as Windows, Linux, Android, and macOS. It also explained how operating systems handle multitasking, process scheduling, file systems, and hardware communication, making them one of the most fundamental components of any computer system.

---

## Linux CLI Basics

Learned the fundamentals of the Linux Command Line Interface (CLI) and how to navigate and interact with the operating system using terminal commands. The room covered essential commands for navigating directories, listing files, creating, copying, moving, renaming, and deleting files and folders, as well as viewing file contents.

It also introduced file permissions, ownership, hidden files, manual pages (`man`), command options, tab completion, command history, and basic utilities used to inspect the system. These concepts are fundamental for working efficiently on Linux systems and form the basis for system administration, scripting, and penetration testing.

---

## Windows CLI Basics

Learned the fundamentals of the Windows Command Line Interface using both **Command Prompt (CMD)** and **PowerShell**. The room covered essential commands for navigating the file system, managing files and directories, viewing system information, and executing common administrative tasks from the terminal.

It also introduced the differences between CMD and PowerShell, environment variables, command help, command history, and basic scripting capabilities. These tools are widely used for Windows administration, automation, troubleshooting, and cybersecurity tasks such as enumeration and system analysis.

---

## Operating System Security

Learned the fundamental security mechanisms used by operating systems to protect users, data, and system resources. The room covered user accounts, authentication, authorization, file permissions, and the principle of least privilege, showing how access to resources is controlled.

It also introduced security features such as updates and patch management, firewalls, antivirus software, encryption, backups, and multi-factor authentication (MFA). The room emphasized that operating system security relies on combining these mechanisms to reduce the attack surface, prevent unauthorized access, and protect against common threats.
